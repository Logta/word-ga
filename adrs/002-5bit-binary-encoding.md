# ADR-002: 遺伝子型を5ビットバイナリエンコーディングで表現

**Status**: Accepted

## Context

遺伝的アルゴリズムでは個体の遺伝子型をどのように表現するかが重要な設計上の決定事項となる。対象文字集合は大文字アルファベット26文字とスペースの計27文字である。文字をそのまま遺伝子として扱う「文字レベル表現」と、バイナリ列として扱う「ビットレベル表現」の2つのアプローチがある。バイナリ表現を採用することで、交叉・突然変異をビット単位で行い、より細かい粒度で進化を観察できる。

## Decision

各文字を5ビットのバイナリ文字列（`'0'`/`'1'` の ASCII 文字）で表現する方式を採用した。文字集合 `"ABCDEFGHIJKLMNOPQRSTUVWXYZ "` のインデックス（0〜26）を5ビット整数として符号化し、A=`00000`、Z=`11001`、スペース=`11010` となる。インデックス 27〜31（`11011`〜`11111`）は範囲外だが、デコード時にスペースとして扱う。

実装は `src/ga/core.ts` の `charToBin`/`binToChar`/`encode`/`decode` 関数で行われ、Wasm に渡す前後でエンコード/デコードを行う。Wasm 側の個体は常にバイナリ文字列として扱われる。

## Consequences

### メリット
- ビット単位の交叉・突然変異が自然に表現できる（1文字の変更が1ビットの反転に対応しうる）
- 適応度計算をビット単位で行えるため、部分的な進化（文字の近似）が評価できる
- Wasm 側では `'0'`/`'1'` の文字比較のみで済むためシンプルな実装になる
- 5ビット固定長のため、文字列長から遺伝子長（× 5）が一意に決まる

### トレードオフ
- エンコード/デコードの変換コストが発生する（ただしオーバーヘッドは小さい）
- 集団の文字列長が文字数の5倍になり、Wasm へ渡すデータ量が増える
- インデックス 27〜31 がすべてスペースにデコードされるため、実際には27通りの表示文字に対して32通りの遺伝子型が存在する（ADR-003 参照）
