# ADR-014: 選択戦略を差し替え可能にし、ルーレット選択を追加

**Status**: Accepted（第3引数の型は ADR-015 で Int → String に変更）

## Context

ADR-007 でエリート選択のみを採用していたが、選択戦略によって収束速度・多様性・局所最適への陥りやすさが大きく変わる。ユーザーが動作の違いを視覚的に比較できるようにするため、選択戦略を切り替え可能にする必要があった。

候補として以下を検討した。

| 戦略 | 特徴 |
|---|---|
| エリート選択（既存） | 上位固定割合から選択。収束は速いが多様性が低下しやすい |
| ルーレット選択 | 適応度比例確率で選択。多様性が高く、初期探索に有効 |
| トーナメント選択 | ランダムに k 個体を抽出し最良を選択。選択圧を k で調整可能 |

トーナメント選択はパラメータ（k）の追加が必要でUIが複雑になるため今回は見送った。

## Decision

選択戦略をエリート・ルーレットの2種類から UI で選択できるようにした。

**MoonBit 側**: `selection_elite.mbt` / `selection_roulette.mbt` に各戦略を分離し、`evolve(population, target, sel: Int)` の第3引数（`0`=エリート、`1`=ルーレット）で切り替える。ルーレット選択は適応度の累積和でホイールを構成し、全適応度が0の場合はランダム選択にフォールバックする。

**TypeScript 側**: `SelectionMethod = "elite" | "roulette"` 型を `types/index.ts` で定義し、`SimState` に `selectionMethod` フィールドを追加。`wasmBridge.wasmEvolve` が文字列を整数に変換して Wasm へ渡す。

**UI 側**: `Controls.tsx` にセレクトボックスを追加。`reset` / `applyTarget` 実行時も選択中の戦略を引き継ぐ。

## Consequences

### メリット
- ユーザーが同一ターゲットで戦略の収束挙動を比較できる
- 各選択戦略がファイル単位で分離され（`selection_elite.mbt` / `selection_roulette.mbt`）、戦略の追加・差し替えが容易
- Wasm API の変更は第3引数の追加のみで、既存の通信プロトコル（ADR-008）に影響なし

### トレードオフ
- ルーレット選択は毎回 O(n) の累積和計算が必要で、エリート選択より世代あたりの計算コストが高い（集団サイズ30では問題なし）
- `evolve` の引数が3つになり、Wasm API のシグネチャが変わったため `mbt.d.ts` の更新が必要だった
- 全個体の適応度が0（初期世代で完全不一致の場合）はルーレットが機能しないため、ランダム選択にフォールバックしている
